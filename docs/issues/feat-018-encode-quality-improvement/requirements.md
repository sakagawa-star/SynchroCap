# feat-018 要求仕様書: Raw→MP4 エンコード品質改善

対象: `tools/raw_tool.py`（`build_ffmpeg_encode_command` 関数）
参照: `docs/issues/feat-003-raw-file-toolkit/design_step3.md`（現在の設計）

---

## 1. プロジェクト概要

### 1.1 何を作るか
`raw_tool.py encode` サブコマンドのffmpegエンコードオプションを変更し、高品質なMP4を出力する。

### 1.2 なぜ作るか
現在のエンコード設定（CBR 2200kbps, preset p4, yuv420p）では産業用カメラの映像に対して画質が不十分。動き解析やキャリブレーション確認に使用するMP4として、ディテールが保持される品質が必要。

### 1.3 誰が使うか
SynchroCapユーザー（録画したRawデータからMP4を生成する場面）。

### 1.4 どこで使うか
NVIDIA GPU搭載のLinux/Windows環境。ffmpegがhevc_nvencエンコーダをサポートしていること。

---

## 2. 用語定義

| 用語 | 定義 |
|------|------|
| Constant QP | 固定量子化パラメータモード。全フレームに同一のQP値を適用する。ビットレートはシーンの複雑さに応じて変動する |
| QP (Quantization Parameter) | 量子化パラメータ。0（最高品質/最大容量）〜51（最低品質/最小容量） |
| CBR | 固定ビットレート。現在の方式 |
| preset | エンコード速度と品質のトレードオフ。p1（最速/最低品質）〜p7（最遅/最高品質） |
| yuv420p | 色差サブサンプリング 4:2:0。色情報が水平・垂直ともに半分 |
| yuv444p | 色差サブサンプリングなし。色情報を完全に保持 |

---

## 3. 機能要求一覧

### FR-001: Constant QPモードによるエンコード

- **機能名**: Constant QP エンコード
- **概要**: CBR方式からConstant QP方式に変更し、一定品質のMP4を出力する
- **入力**: 変更なし（既存CLIと同一）
- **出力**: CQモードでエンコードされたMP4ファイル
- **受け入れ基準**: ffmpegに `-rc constqp -qp 20` が渡され、MP4が生成される

### FR-002: 高品質presetの適用

- **機能名**: preset p7 の適用
- **概要**: エンコードpresetを p4（medium）から p7（slowest/best quality）に変更する
- **入力**: 変更なし
- **出力**: p7 preset でエンコードされたMP4
- **受け入れ基準**: ffmpegに `-preset p7` が渡される

### FR-003: yuv444p 色空間の使用

- **機能名**: YUV 4:4:4 色空間
- **概要**: Bayer demosaic後の色情報劣化を抑制するため、yuv444pを使用する
- **入力**: 変更なし
- **出力**: yuv444p でエンコードされたMP4
- **受け入れ基準**: ffmpegに `-vf format=yuv444p` が渡される。HEVC profile は `rext`（Range Extensions）が使用される

### FR-004: CLIオプションによるQP値の指定（任意）

- **機能名**: QP値のCLI指定
- **概要**: `--qp` オプションでQP値を指定可能にする。デフォルトは20
- **入力**: `--qp <int>` （0〜51、デフォルト20）
- **出力**: 指定QP値でエンコードされたMP4
- **受け入れ基準**: `--qp 25` を指定した場合、ffmpegに `-qp 25` が渡される。省略時は `-qp 20` が渡される

## 4. 非機能要求

### 4.1 パフォーマンス

- エンコード速度は現在より低下することを許容する（preset p7 のため）
- オフラインツールであり、リアルタイム性は不要

### 4.2 互換性

- 出力MP4はVLCおよび一般的な動画プレイヤーで再生可能であること
- yuv444p + HEVC rext profile は一部の古いプレイヤーで非対応の可能性がある。これは許容する

### 4.3 ファイルサイズ

- 現在の 2.2Mbps 固定から大幅に増加する（シーン依存で 10〜50Mbps 程度）。これを許容する

---

## 5. 制約条件

- ffmpeg が hevc_nvenc エンコーダをサポートしていること（NVIDIA GPU必須）
- hevc_nvenc が yuv444p ピクセルフォーマットをサポートしていること（確認済み: Supported pixel formats に yuv444p を含む）
- フレーム選択ロジック（`build_frame_plan`, `encode_frames`）は変更しない

---

## 6. 優先順位

| ID | 優先度 | 備考 |
|----|--------|------|
| FR-001 | Must | 画質改善の主要因 |
| FR-002 | Must | 品質最適化 |

### MVP範囲
FR-001 + FR-002（Constant QPモード + p7 preset）。これだけで画質は大幅に改善する。
