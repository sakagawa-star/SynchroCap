# feat-021: ChArUco Board Capture Guide（ChArUco ボード撮影ガイド）

## Status

Open（参考資料整備済み。アプリケーションへの取り込み方は未定）

## 概要

intrinsics キャリブレーション用の ChArUco ボード撮影ガイドを SynchroCap に取り込む案件。

2026-06-10 の調査で、**RMS 再投影誤差が良好（0.46px）でも、撮影ポーズの多様性不足（特にボードの傾き不足）により焦点距離 fx が決まらない**問題が確認された（下記「背景」）。撮影方法を改善することで問題が解決することも実機で実証済み。

本案件は2段階で進める:

1. **撮影ガイドの参考資料化（完了）** → [capture_guide.md](capture_guide.md)
2. **アプリへの取り込み（未定・未着手）** — 取り込み方の候補は capture_guide.md 末尾の「ガイダンス機能化する場合の実装要素」を参照。着手時に要求仕様書（requirements.md）・機能設計書（design.md）を作成する

## 背景（発端となった問題）

- `tools/offline_calibration.py` で得た intrinsics（cam41520554, 88枚, RMS 0.4614px）を使って外部パラメーター推定（基準点方式・別リポジトリの独自ツール）を行ったところ、再投影誤差 RMSE 142.74px と破綻した
- 調査の結果、キャリブレーション計算・エクスポートは正常で、**撮影データの幾何条件（ボードがほぼ正対・距離ほぼ一定）が原因で fx が原理的に決定できない状態**だったことが判明（fx を 9% 変えて固定しても RMS が +0.05px しか悪化しない = データが fx を区別できない）
- ボードを傾けて撮り直したところ（50枚, 傾き 75%点 36°）、fx は 1112.6 → 1216.9 に変化し、独立した基準点推定（fx≈1210.5）と 0.5% で一致。外部パラメーター推定 RMSE は 142.74 → 4.84px に改善
- 既存のカバレッジヒートマップ（feat-010）は「画面内の位置」のみを可視化しており、この問題（傾き・距離の多様性不足）を検出できない

詳細な経緯・実証データは [capture_guide.md](capture_guide.md) を参照。
