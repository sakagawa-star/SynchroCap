# 要求仕様書: Camera Calibration - Calibration Calculation + Result Display

対象: feat-011
作成日: 2026-03-13
基準文書: `docs/REQUIREMENTS_STANDARD.md`

---

## 1. プロジェクト概要

### 1.1 何を作るか

feat-009で蓄積されたキャプチャデータ（image_points, object_points）を使い、`cv2.calibrateCamera()` でカメラ内部パラメータ（カメラ行列、歪み係数）を算出する機能と、算出結果をCalibrationタブ（Tab5）のUI上に表示する機能を追加する。

### 1.2 なぜ作るか

キャリブレーション計算はカメラの内部パラメータ取得の中核処理であり、後続のエクスポート（feat-012）やPose2Simワークフローの前提条件である。計算結果とキャプチャごとの再投影誤差を表示することで、ユーザーがキャリブレーション品質を評価し、品質の悪いキャプチャを除去して再計算する反復的な改善ワークフローを実現する。

### 1.3 誰が使うか

SynchroCapを使用してモーションキャプチャ用の同期録画を行うオペレーター。

### 1.4 どこで使うか

SynchroCapと同一のPC環境（Ubuntu Linux、micromamba SynchroCap環境）。カメラ（DFK33GR0234）が物理接続された状態で使用する。

## 2. 用語定義

| 用語 | 定義 |
|------|------|
| カメラ行列 | 3×3の内部パラメータ行列。焦点距離（fx, fy）と主点（cx, cy）を含む |
| 歪み係数 | レンズ歪みを表す係数ベクトル。(k1, k2, p1, p2, k3) の5パラメータ |
| RMS再投影誤差 | `cv2.calibrateCamera()` の戻り値。全キャプチャ・全コーナーにわたる再投影誤差のRMS値（ピクセル単位）。値が小さいほどキャリブレーション品質が高い |
| キャプチャごとの再投影誤差 | 個々のキャプチャにおける再投影誤差のRMS値（ピクセル単位）。`cv2.projectPoints()` で算出する |
| image_points | 画像上で検出されたコーナーの2D座標。shape=(N,1,2), float32 |
| object_points | ボード上のコーナーの3D座標。shape=(N,1,3), float32 |
| rvecs | 各キャプチャの回転ベクトル。`cv2.calibrateCamera()` の出力 |
| tvecs | 各キャプチャの並進ベクトル。`cv2.calibrateCamera()` の出力 |

## 3. 機能要求一覧

### FR-001: キャリブレーション計算

- **概要**: 蓄積されたキャプチャデータから `cv2.calibrateCamera()` を実行し、カメラ内部パラメータを算出する
- **入力**:
  - `captures: list[CaptureData]` — キャプチャリスト（4件以上）
  - `image_size: tuple[int, int]` — 画像サイズ (width, height)
- **前提条件**: キャプチャが4件以上存在すること
- **処理**:
  1. 全キャプチャの `object_points` と `image_points` をリスト形式に変換する
  2. `cv2.calibrateCamera(objectPoints, imagePoints, imageSize, None, None)` を実行する
  3. 戻り値からRMS再投影誤差、カメラ行列、歪み係数、rvecs、tvecsを取得する
  4. キャプチャごとの再投影誤差を `cv2.projectPoints()` で算出する
- **出力**: CalibrationResult（RMS再投影誤差、カメラ行列、歪み係数、rvecs、tvecs、キャプチャごとの再投影誤差）
- **受け入れ基準**:
  - キャプチャ4件以上で「Calibrate」ボタンを押すとキャリブレーション計算が実行される
  - 計算結果（RMS誤差、カメラ行列パラメータ、歪み係数）がUI上に表示される
  - 計算が正常完了した場合、結果が保持される
- **異常系**:
  - `cv2.calibrateCamera()` が `cv2.error` を送出した場合: ステータスラベルに `Calibration failed: {エラーメッセージ}` と表示し、結果は `None` のまま保持する
  - キャプチャが4件未満の場合: 「Calibrate」ボタンが無効化されており、計算は実行されない

### FR-002: 結果表示

- **概要**: キャリブレーション計算結果をCalibrationタブの左パネルに表示する
- **入力**: CalibrationResult（FR-001の出力）
- **処理**:
  1. 「Results」QGroupBoxに以下の値を表示する:
     - RMS Error: RMS再投影誤差（小数点以下4桁、単位: px）
     - fx: 焦点距離X（小数点以下1桁）
     - fy: 焦点距離Y（小数点以下1桁）
     - cx: 主点X（小数点以下1桁）
     - cy: 主点Y（小数点以下1桁）
     - Dist: k1, k2, p1, p2, k3（各値を小数点以下4桁、`k1=..., k2=...\np1=..., p2=...\nk3=...` の3行表示。左パネル幅200pxに収めるため改行する）
  2. 結果が存在しない場合（計算前、またはクリア後）は各フィールドに `---` を表示する
- **出力**: UI上の結果表示の更新
- **受け入れ基準**:
  - 「Results」グループに上記すべての値が表示される
  - 結果未算出時は `---` が表示される
  - 数値のフォーマットが上記の桁数で統一されている

### FR-003: キャプチャごとの再投影誤差表示

- **概要**: キャリブレーション計算後、キャプチャリストの各エントリに再投影誤差を追記表示する
- **入力**: CalibrationResult.per_image_errors（FR-001の出力の一部）
- **処理**:
  1. キャリブレーション計算後、キャプチャリスト（QListWidget）の各エントリの表示を更新する
  2. 表示形式: `#01: 24 corners | err: 0.32 px`（再投影誤差を小数点以下2桁で付加。単位 px の前にスペースを入れる）
  3. キャリブレーション結果がクリアされた場合、元の表示形式 `#01: 24 corners` に戻す
- **出力**: キャプチャリストの表示更新
- **受け入れ基準**:
  - キャリブレーション計算後、各キャプチャエントリに再投影誤差が表示される
  - 誤差の大きいキャプチャ（品質が悪い）を視覚的に特定できる
  - 結果クリア後は元の表示形式に戻る

### FR-004: Calibrateボタンと結果のライフサイクル管理

- **概要**: キャリブレーション計算の実行ボタンと、結果の有効/無効を管理する
- **入力**: ユーザーの「Calibrate」ボタンクリック、キャプチャの追加/削除
- **処理**:
  1. 「Calibrate」ボタンはキャプチャが4件以上の場合のみ有効化される
  2. キャプチャが追加または削除された場合、キャリブレーション結果をクリアする（結果表示を `---` に戻し、キャプチャリストの再投影誤差表示を除去する）
  3. 「Calibrate」ボタンを押すとキャリブレーション計算を実行し、結果を表示する
  4. カメラ切替・タブ離脱時は既存の `stop_live_view()` でキャプチャがクリアされるため、結果も自動的にクリアされる
- **出力**: ボタン状態の更新、結果のクリア/表示
- **受け入れ基準**:
  - キャプチャ3件以下では「Calibrate」ボタンが無効
  - キャプチャ4件以上で「Calibrate」ボタンが有効
  - キャプチャを追加/削除すると結果がクリアされる
  - 結果クリア後に「Calibrate」ボタンを押すと再計算される
  - カメラ切替・タブ離脱後は結果がクリアされている

## 4. 非機能要求

### 4.1 パフォーマンス

| 項目 | 基準 |
|------|------|
| キャリブレーション計算 | HD解像度、30キャプチャで1秒以下。`cv2.calibrateCamera()` はCPU処理だが、単一カメラの内部パラメータ推定は計算量が小さい |
| 結果表示更新 | 計算完了後、即座にUI上に反映される（QLabel.setText() のみ） |
| ライブビューへの影響 | キャリブレーション計算はメインスレッドで同期実行するが、1秒以内に完了するためライブビューの一時停止は許容する |

### 4.2 信頼性

キャリブレーション結果はメモリ上にのみ保持される。永続化は後続案件（feat-012: Export、feat-013: Session Save）で対応する。

### 4.3 ログ出力

Python標準の `logging` モジュールを使用する。ログ出力の詳細（レベル、出力ポイント）は機能設計書セクション8で定義する。

### 4.4 対応環境

feat-009/010と同一。Ubuntu Linux、micromamba SynchroCap環境。

## 5. 制約条件

### 5.1 使用必須ライブラリ

feat-009/010と同一。新規ライブラリの追加なし。`cv2.calibrateCamera()`, `cv2.projectPoints()` は既存ライブラリの機能。

### 5.2 カメラ設定変更禁止

CLAUDE.md「カメラ設定変更禁止ルール」に従う。Calibrationタブはカメラ設定を変更しない。

### 5.3 入力データ形式

feat-009で定義された `CaptureData` をそのまま使用する:

- `image_points`: numpy.ndarray, shape=(N,1,2), float32
- `object_points`: numpy.ndarray, shape=(N,1,3), float32
- `image_size`: tuple[int, int] (width, height) — `_capture_image_size`

`CaptureData` のフィールドは変更しない。

### 5.4 最小キャプチャ数

キャリブレーション計算に必要な最小キャプチャ数は4件とする。`cv2.calibrateCamera()` は理論上2件以上で動作するが、パラメータの安定推定のため4件を最小とする。

## 7. スコープ外

以下は本案件の対象外とする（後続案件で対応）:

- キャリブレーション結果のファイルエクスポート（feat-012）
- セッション保存/再開（feat-013）
- 歪み補正済みプレビュー（undistort表示）
- fisheyeモデル対応
- 固定アスペクト比制約（`cv2.CALIB_FIX_ASPECT_RATIO` フラグ）
- Board Settings 変更後の既存キャプチャデータの無効化（現行では Board Settings 変更時にキャプチャはクリアされないため、古いキャプチャデータでキャリブレーションが実行される。これはfeat-013のセッション管理で対応する）
- 外部パラメータ（extrinsics）の算出
- キャリブレーション品質の自動判定（RMS閾値によるOK/NG判定）
- 結果の可視化（再投影マップ、歪み分布の描画）
