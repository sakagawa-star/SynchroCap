# 要求仕様書: Camera Calibration - Auto Capture (Stability Trigger)

対象: feat-009
作成日: 2026-03-05
基準文書: `docs/REQUIREMENTS_STANDARD.md`

---

## 1. プロジェクト概要

### 1.1 何を作るか

feat-008で作成したCalibrationタブ（Tab5）に、安定検出トリガーによる自動キャプチャ機能を追加する。ボード検出が一定時間連続で成功（ボードが静止している）した場合に自動でキャプチャし、検出結果（image_points, object_points）をメモリ上に蓄積する。オプションで静止画（PNG）をファイル保存する。

### 1.2 なぜ作るか

キャリブレーション用ボードは両手で持つ必要があるため、PC操作（ボタンクリック）による手動キャプチャは不可能である。ボードの安定検出を自動判定し、ハンズフリーでキャプチャを行えるようにする。

### 1.3 誰が使うか

SynchroCapを使用してモーションキャプチャ用の同期録画を行うオペレーター。

### 1.4 どこで使うか

SynchroCapと同一のPC環境（Ubuntu Linux、micromamba SynchroCap環境）。カメラ（DFK33GR0234）が物理接続された状態で使用する。

---

## 2. 用語定義

| 用語 | 定義 |
|------|------|
| 安定検出 | ボード検出が一定時間（安定判定時間）連続で成功し続けている状態 |
| 安定判定時間 | 安定検出と判定するまでの連続成功時間。2.0秒 |
| 自動キャプチャ | 安定検出が成立した時点で自動的にキャプチャを実行する動作 |
| クールダウン | 自動キャプチャ後、次の安定判定を開始するまでの待機時間。3.0秒 |
| キャプチャ | ライブビューの1フレームからボード検出結果（image_points, object_points）を保存する操作 |
| image_points | 画像上で検出されたコーナーの2D座標。shape=(N,1,2), float32 |
| object_points | ボード上のコーナーの3D座標。shape=(N,1,3), float32 |
| キャプチャセッション | カメラ接続からカメラ切替またはタブ離脱までの一連のキャプチャ操作 |

## 3. 機能要求一覧

### FR-001: 安定検出トリガーによる自動キャプチャ

- **概要**: ライブビュー中にボード検出が安定判定時間（2.0秒）連続で成功した場合、自動的にキャプチャを実行する
- **入力**: ライブビューのフレームストリーム（ユーザー操作なし）
- **前提条件**: ライブビューが動作中であること
- **処理**:
  1. 毎フレームのボード検出結果を監視する
  2. 検出成功が連続で2.0秒以上継続した場合、安定検出と判定する
  3. 安定検出が成立した時点で、直近の検出結果（image_points, object_points）をキャプチャリストに追加する
  4. 安定判定は経過時間ベースで行う（`time.monotonic()`で連続成功開始時刻を記録し、現在時刻との差分が安定判定時間を超えたか判定する）
  5. 検出失敗が1フレームでも発生した場合、連続成功開始時刻をリセットする
- **出力**:
  - 検出結果（image_points, object_points, charuco_ids, num_corners）がメモリ上のキャプチャリストに追加される
  - キャプチャ数カウンタが更新される
  - キャプチャリスト（QListWidget）に新しいエントリが表示される（例: `#01: 24 corners`）
  - image_sizeは最初のキャプチャから取得し、以降のキャプチャで一致を検証する
- **受け入れ基準**:
  - ボードを静止させて2.0秒以上経過すると自動的にキャプチャされる
  - ボードを動かしている間（検出失敗が混在）はキャプチャされない
  - キャプチャ成功時にステータスラベルに `Captured #N (X corners)` と表示される
  - キャプチャはメモリ上に保持される
- **異常系**:
  - image_sizeが最初のキャプチャと異なる場合: キャプチャを拒否し、ステータスに `Image size mismatch` と表示する

### FR-002: クールダウン制御

- **概要**: 自動キャプチャ後、同じ位置での連続キャプチャを防止するためにクールダウン期間を設ける
- **入力**: 自動キャプチャの実行完了
- **処理**:
  1. 自動キャプチャ成功直後にクールダウン状態に遷移する
  2. クールダウン時間は3.0秒（固定値）
  3. クールダウン中はボード検出処理とオーバーレイ描画は継続するが、安定判定カウントは停止する
  4. クールダウン終了後、連続成功開始時刻をリセットして安定判定を再開する
- **出力**:
  - クールダウン中はステータスラベルに残り時間を表示する（例: `Cooldown: 2.1s`）
- **受け入れ基準**:
  - キャプチャ直後にボードを静止させ続けても、3.0秒間は次のキャプチャが発生しない
  - クールダウン中もライブビューとボード検出オーバーレイは正常に表示される
  - クールダウン終了後、ボードが静止し続けていれば2.0秒後に次のキャプチャが発生する

### FR-003: キャプチャリスト管理

- **概要**: キャプチャ済みデータの一覧表示と削除機能を提供する
- **入力**:
  - 「Delete」ボタンクリック: キャプチャリストで選択中のエントリを削除する
  - 「Clear All」ボタンクリック: 全キャプチャを削除する
- **出力**: キャプチャリストの更新、キャプチャ数カウンタの更新
- **受け入れ基準**:
  - キャプチャリスト（QListWidget）に各キャプチャが `#01: 24 corners`、`#02: 22 corners` の形式で表示される
  - 「Delete」ボタンはキャプチャリストでアイテムが選択されている場合のみ有効化される
  - 「Clear All」ボタンはキャプチャが1件以上ある場合のみ有効化される
- **境界条件**:
  - キャプチャ0件: 「Delete」「Clear All」ボタンが無効化される
  - 全削除後: キャプチャリストが空になる

### FR-004: キャプチャ時フィードバック

- **概要**: 自動キャプチャの発生と安定検出の進行状況をユーザーに視覚的に伝える
- **入力**: 安定判定の進行状況、キャプチャの発生
- **出力**:
  - **安定検出中**: ステータスラベルに安定判定の進行状況を表示する（例: `Stability: 1.2s / 2.0s`）
  - **キャプチャ成功時**: ライブビュー領域（QLabel）の枠線色を一瞬（300ms）緑色に変更するフラッシュ演出を行う
  - **クールダウン中**: ステータスラベルにクールダウン残り時間を表示する（例: `Cooldown: 2.1s`）
- **受け入れ基準**:
  - ボードを静止させ始めると、ステータスに安定判定の進行が表示される
  - キャプチャが発生すると、ライブビューの枠が緑色にフラッシュする
  - フラッシュは300ms後に元のスタイルに戻る

### FR-005: 静止画保存（デバッグ用）

- **概要**: 自動キャプチャ時に、検出オーバーレイ付きの静止画をPNGファイルとして保存する。UI上のチェックボックスでON/OFFを切り替えられる
- **入力**: 自動キャプチャの発生、Save Images チェックボックスの状態
- **前提条件**: Save Images チェックボックスがONであること
- **処理**:
  1. キャプチャセッション開始時（最初のキャプチャ実行時）にタイムスタンプディレクトリを作成する
  2. 保存パス: `captures/YYYYMMDD-HHMMSS/intrinsics/cam{serial}/capture_{番号:03d}.png`
     - `YYYYMMDD-HHMMSS`: キャプチャセッション開始時のタイムスタンプ（`datetime.now().strftime("%Y%m%d-%H%M%S")`）
     - `{serial}`: カメラのシリアル番号
     - `{番号:03d}`: 1始まりの3桁ゼロパディング（例: 001, 002, 003）
  3. 保存する画像は検出オーバーレイ描画済みのBGRフレーム
  4. `cv2.imwrite()` でPNG形式で保存する
- **出力**: PNGファイルの保存
- **受け入れ基準**:
  - Save ImagesがOFFの場合、ファイル保存は行われない
  - Save ImagesがONの場合、キャプチャのたびにPNGファイルが保存される
  - ディレクトリが存在しない場合は自動作成される（`mkdir(parents=True, exist_ok=True)`）
  - 保存パスがMulti Viewの録画保存先規約（`captures/YYYYMMDD-HHMMSS/`）に準拠している
- **異常系**:
  - ファイル保存に失敗した場合（ディスク容量不足、権限エラー）: ステータスに `Save failed: {error}` と表示する。キャプチャ自体（メモリ上のデータ）は正常に保持される。保存失敗でキャプチャが中止されることはない
- **デフォルト**: OFF

### FR-006: カメラ切替・タブ離脱時のキャプチャクリア

- **概要**: カメラを切り替えた場合またはタブから離脱した場合、キャプチャリストをクリアする
- **入力**: ユーザーが別のカメラをクリック、または別タブに切り替え
- **出力**: キャプチャリスト・安定判定状態・キャプチャセッションが全てクリアされる
- **受け入れ基準**:
  - 別カメラ選択時にキャプチャリストが空になる
  - タブ離脱時にキャプチャリストが空になる
  - 同一カメラを再クリックした場合はクリアしない（feat-008の既存動作: 同一カメラは無視）
  - クリア後に再びキャプチャを開始した場合、新しいタイムスタンプディレクトリが使用される

## 4. 非機能要求

### 4.1 パフォーマンス

| 項目 | 基準 |
|------|------|
| 安定判定処理 | フレーム処理（33ms間隔）を遅延させない。時刻比較のみで追加コストは無視可能 |
| 静止画保存 | `cv2.imwrite()` は同期実行で十分。1枚あたり50ms以下（HD解像度のPNG） |
| ライブビューへの影響 | キャプチャ・保存処理中もライブビューが停止しない |

### 4.2 信頼性

キャプチャデータはメモリ上にのみ保持される。アプリ終了時やカメラ切り替え時にデータが消失する。永続化は後続案件（feat-013: Session Save/Resume）で対応する。静止画保存（FR-005）はデバッグ用であり、保存失敗時もキャプチャ処理に影響しない。

### 4.3 ログ出力

feat-008と同じ `logging` モジュールを使用する。

### 4.4 対応環境

feat-008と同一。

## 5. 状態遷移

feat-008の状態（Idle, Connecting, LiveView）を維持する。LiveView状態の内部に安定判定の状態を追加する。

### 5.1 CalibrationWidgetの状態

| 状態 | 説明 |
|------|------|
| Idle | カメラ未接続（feat-008と同一） |
| Connecting | カメラ接続中（feat-008と同一） |
| LiveView | ライブビュー表示中。安定検出・自動キャプチャが動作する |

### 5.2 安定判定の内部状態（LiveView内）

| 状態 | 説明 |
|------|------|
| Monitoring | ボード検出結果を監視中。連続成功を追跡している |
| Cooldown | キャプチャ後のクールダウン中。安定判定を停止している |

遷移:
- `Monitoring → Monitoring`: 検出失敗（連続成功開始時刻をリセット）
- `Monitoring → Cooldown`: 安定判定時間到達（キャプチャを実行してクールダウン開始）
- `Cooldown → Monitoring`: クールダウン時間経過（連続成功開始時刻をリセット）

LiveViewからIdle/Connectingに遷移する場合、安定判定状態もリセットされる。

## 6. 制約条件

### 6.1 使用必須ライブラリ

feat-008と同一。新規ライブラリの追加なし。

### 6.2 カメラ設定変更禁止

feat-008と同一。Calibrationタブはカメラ設定を変更しない。

### 6.3 データ保持形式

キャプチャデータは以下の形式でメモリ上に保持する。この形式はfeat-010（ヒートマップ）およびfeat-011（キャリブレーション計算）で参照される:

- `image_points`: numpy.ndarray, shape=(N,1,2), float32
- `object_points`: numpy.ndarray, shape=(N,1,3), float32
- `charuco_ids`: numpy.ndarray | None, shape=(N,1), int32
- `num_corners`: int
- `image_size`: tuple[int, int] (width, height)

## 8. スコープ外

以下は本案件の対象外とする（後続案件で対応）:

- キャリブレーション計算（`cv2.calibrateCamera()`）と結果表示（feat-011）
- カバレッジヒートマップ（feat-010）
- キャリブレーション結果のファイルエクスポート（feat-012）
- セッション保存/再開（キャプチャデータの永続化）（feat-013）
- fisheyeモデル対応
- キャプチャ画像のサムネイル表示
- キャプチャ時の品質チェック（ブレ検出、ボードサイズ判定）
- 音声フィードバック
- 安定判定時間・クールダウン時間のUI上での変更
