# feat-017: Live View During Recording 要求仕様書

## 1. プロジェクト概要

- **何を作るのか**: 録画中もマルチビュー（Tab3）のライブビューを継続表示する機能
- **なぜ作るのか**: 録画中にカメラ映像を確認できず、撮影対象のフレーミングや異常検出ができない
- **誰が使うのか**: SynchroCap ユーザー（複数カメラ同時録画を行う研究者）
- **どこで使うのか**: メインアプリの Multi View タブ（Tab3）

## 2. 用語定義

| 用語 | 定義 |
|------|------|
| DisplayWidget | IC4 SDK が提供する PySide6 ウィジェット。Bayer 変換 + 画面描画を自動実行する |
| Display | `DisplayWidget.as_display()` で取得するオブジェクト。`stream_setup()` の引数として使用 |
| stream_setup | IC4 Grabber のメソッド。sink, display, setup_option を受け取り、データストリームを確立する |
| DEFER_ACQUISITION_START | `stream_setup()` のオプション。ストリーム確立後、acquisition 開始を遅延させる（Action Scheduler による同期開始に使用） |
| RecordingSlot | 録画中の各カメラの状態を保持するデータクラス |

## 3. 機能要求一覧

### FR-001: 録画中のライブビュー継続

- **機能名**: 録画中ライブビュー
- **概要**: 録画中も各カメラの DisplayWidget にライブ映像が表示される
- **入力**: 録画開始操作（既存の Start ボタン）
- **出力**: 録画中、DisplayWidget にカメラ映像がリアルタイムで表示される
- **受け入れ基準**:
  - 録画中に全カメラのライブビューが表示されている
  - 録画ファイル（MP4/Raw）にフレームが正常に書き込まれている
  - フレーム数が display なしの場合と同等であること
  - CSV タイムスタンプが正常に記録されていること

### FR-002: 録画開始・停止時の一時的途切れの許容

- **機能名**: 映像途切れの許容
- **概要**: 録画開始時と停止時に `stream_stop()` → `stream_setup()` の間、ライブビューが一時的に途切れる
- **入力**: 録画開始または停止操作
- **出力**: 一時的な映像途切れの後、ライブビューが復帰する
- **受け入れ基準**:
  - 録画開始時: `stream_stop()` から録画用 `stream_setup()` 完了までの間、映像が途切れる
  - 録画停止時: 録画用ストリーム停止からプレビュー用 `stream_setup()` 完了までの間、映像が途切れる
  - 途切れの前後でアプリがクラッシュしない

### FR-003: 既存録画機能への影響なし

- **機能名**: 録画機能の後方互換性
- **概要**: 録画の開始・停止・フレーム書き込み・CSV 記録・エラーハンドリングの動作が変わらない
- **入力**: 既存の録画操作
- **出力**: 既存と同一の録画ファイル出力
- **受け入れ基準**:
  - MP4/Raw の出力ファイルが既存と同一フォーマット
  - CSV タイムスタンプ記録が既存と同一
  - Action Scheduler による同期開始が既存と同一の挙動

## 4. 非機能要求

- **パフォーマンス**: Display への描画は IC4 SDK 内部で録画ワーカースレッドとは独立して行われるため、録画ワーカースレッドの処理には影響しない。録画中の FPS（フレーム取得レート）が display なしの場合と同等であること
- **信頼性**: Display への描画失敗が録画処理に影響を与えてはならない（IC4 SDK が内部で分離管理）

## 5. 制約条件

- IC4 SDK の `stream_setup(sink, display)` が sink と display の同時接続をサポートしていること（確認済み）
- `DEFER_ACQUISITION_START` オプションが display 付きでも正常動作すること（手動テストで確認が必要）
- 録画スレッド（`_worker_mp4` / `_worker_raw`）のフレーム取得ロジックは変更しない

| ID | 優先度 | 備考 |
|----|--------|------|
| FR-001 | Must | 本案件の主要機能 |
| FR-002 | Must | 設計上の制約として明示 |
